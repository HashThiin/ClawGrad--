# 03_分数超上限问题修复

## 日期
2026-05-20

## 问题描述
测试使用 `qwen3.6-flash` 模型，流水线全部通过（4 阶段均成功），但日志显示：

```
FeedbackStage ok: score=122.0/100.0, items=7
```

**总分 122.0 超过了满分 100.0**，用户界面上会显示"122.0/100.0 分"，这显然不合理。原因是 AI 在计算总分时出错（各题得分汇总后超过 100），但系统没有做分数上限校验。

## 修复方案

### 白话方案
AI 算分不可靠，经常算出超满分的结果。在存入结果之前加一道校验：
- 如果 AI 给的总分超过 100，就把它改成 100
- 如果某道题的分数超过该题满分，也截断
- 截断时记录日志，方便知道 AI 算出过什么离谱分数

### 代码解释

**分数上限校验方法**

```java
// FeedbackStage.java - sanitizeScores()
// 在 execute() 中，设置结果之前调用
private void sanitizeScores(AssignmentGradingResult result) {
    Double max = result.getMaxScore();
    if (max == null) return;
    
    // 1. 限制总分
    if (result.getTotalScore() != null && result.getTotalScore() > max) {
        log.warn("totalScore {} capped to maxScore {}", result.getTotalScore(), max);
        result.setTotalScore(max);
    }
    
    // 2. 限制各题分数
    if (result.getItems() != null) {
        for (ItemGradingResult item : result.getItems()) {
            Double itemMax = item.getMaxScore();
            if (item.getScore() != null && itemMax != null && item.getScore() > itemMax) {
                log.warn("item[{}] score {} capped to maxScore {}", 
                    item.getIndex(), item.getScore(), itemMax);
                item.setScore(itemMax);
            }
        }
    }
}
```

**调用时机：在 execute() 中的定位**

```java
// execute() 方法中：
// Step 1: totalScore 兜底（AI 没返回总分时由 items 汇总）
if (result.getTotalScore() == null && result.getItems() != null) {
    double sum = 0;
    for (ItemGradingResult ir : result.getItems()) {
        if (ir.getScore() != null) sum += ir.getScore();
    }
    result.setTotalScore(sum);
}

// Step 2: 分数上限校验（新增）
sanitizeScores(result);

// Step 3: 设置结果并完成
ctx.setResult(result);
taskStore.completeTask(ctx.getTaskId(), result);
```
